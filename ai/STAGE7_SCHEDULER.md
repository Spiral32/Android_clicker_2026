# Stage 7 — Scheduler

## Цель этапа
Реализовать систему планирования задач для автоматизации сценариев в Prog Set Touch.

## Архитектурные решения
- Планировщик должен быть интегрирован в существующую state machine (StateMachine)
- Использовать Android AlarmManager для точного планирования
- Хранить расписания в persistent storage (SharedPreferences или Room)
- Интегрировать с системными событиями через BroadcastReceiver

## Компоненты для реализации

### 1. Scheduler Domain Model
- `Schedule` entity: время, дни недели, сценарий для запуска
- `SchedulerState`: активные/неактивные расписания
- `SchedulerEvent`: события планировщика (schedule added, removed, triggered)

### 2. Platform Layer (Kotlin)
- `SchedulerManager`: управление расписаниями через AlarmManager
- `ScheduleStorage`: сохранение/загрузка расписаний
- `SystemEventReceiver`: обработка системных событий (boot, time change)

### 3. Flutter UI
- `SchedulerScreen`: список расписаний, добавление/редактирование
- `ScheduleForm`: форма для создания расписания
- `SchedulerBloc`: управление состоянием планировщика

## Задачи реализации

### Задача 1: Domain Model ✅
- Создать `Schedule` класс в domain layer
- Определить типы расписаний (одноразовые, повторяющиеся)
- Интегрировать с существующими сценариями

### Задача 2: Platform Storage ✅
- Реализовать `ScheduleStorage` для сохранения расписаний
- Обеспечить миграцию данных при обновлениях

### Задача 3: AlarmManager Integration ✅
- Создать `SchedulerManager` для установки/отмены alarm
- Обработать точность и battery optimization

### Задача 4: System Events ✅
- Реализовать `SystemEventReceiver` для восстановления расписаний после перезагрузки
- Обработать изменения времени/часового пояса

### Задача 5: UI Implementation ✅
- Создать `SchedulerScreen` с Material Design
- Реализовать формы для создания расписаний

### Задача 6: Integration with Execution Engine ✅

- Обеспечить безопасность (проверка разрешений перед запуском)

### Задача 7: Testing and Documentation ✅

## Критерии завершения

- Сценарии запускаются автоматически по расписанию
