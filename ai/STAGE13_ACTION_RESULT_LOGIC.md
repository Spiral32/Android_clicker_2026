# Stage 13 — Логика «Действие → Результат»
Владелец: AI Agent
---

## Цель
Закрепить и документировать логику «Действие → Результат», которая уже реализована в проекте:
- При записи в режиме Step-by-step capture: после каждого действия можно сохранить эталонный скриншот результата
- При выполнении сценария: выполнить действие, дождаться появления экрана (с учетом таймаута и порога схожести), сравнить с эталоном
- Если экран совпал — шаг пройден; если нет — обработать ошибку согласно continueOnFailure

---

## Правила продукта (уже реализованы)

1. **Глобальная настройка**: `globalVerificationEnabled` в Настройках управляет всеми визуальными проверками.
2. **Настройка шага**: `verificationEnabled` для каждого шага (по умолчанию отключена).
3. **Запись**:
   - В режиме Step-by-step capture (POINT_CAPTURE) есть кнопка "Screenshot" (только если globalVerificationEnabled = true)
   - Кнопка "Screenshot" сохраняет скриншот для **предыдущего** записанного действия
   - При остановке записи автоматически делается финальный скриншот для последнего действия
4. **Выполнение**:
   - Если verificationEnabled = true и есть resultImageFileName — запускается цикл проверки
   - Цикл проверки: опрос каждую 1 секунду, до timeoutMs (от 1 сек до 5 минут)
   - Сравнение: по гистограмме (ComparisonMethod.HISTOGRAM) с порогом thresholdPercent (1..100%)
   - FLAG_SECURE: если захват экрана вернул null из-за FLAG_SECURE — верификация считается успешной
5. **Обработка ошибок**:
   - continueOnFailure = false: остановить текущий сценарий как упавший
   - continueOnFailure = true: пометить текущий сценарий как упавший и перейти к следующему

## Техническое состояние (уже реализовано)

- **RecorderManager.kt**: кнопка "Screenshot", функция captureScreenshotForPreviousAction()
- **ExecutionEngine.kt**: цикл верификации с опросом каждую секунду, сравнение по гистограмме
- **ScreenshotVerifier.kt**: сравнение скриншотов, проверка FLAG_SECURE
- **ScreenshotStorageManager.kt**: сохранение и загрузка скриншотов
- **ScenarioStepEditorDialog.dart**: UI для настройки verificationEnabled, thresholdPercent, timeoutMs, continueOnFailure
