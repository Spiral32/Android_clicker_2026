# 🤖 Prog Set Touch (Умный Автокликер)

**Prog Set Touch** — это продвинутый инструмент автоматизации действий на экране вашего Android-устройства. В отличие от обычных автокликеров, которые работают "вслепую", это приложение наделено «зрением»: оно умеет анализировать происходящее на экране, понимать, успешно ли выполнено действие, и гибко реагировать на непредвиденные ситуации.

Приложение идеально подходит для автоматизации рутинных задач, тестирования интерфейсов и написания сложных макросов.

---

## ✨ Ключевые возможности

### 1. 🎛 Мощный движок выполнения жестов
Все взаимодействия происходят на системном уровне с помощью Accessibility Service (Службы специальных возможностей), что имитирует реальные касания пользователя.
*   **Доступные жесты:** Обычный клик (Tap), Двойной клик (Double Tap), Долгое нажатие (Long Press), Свайпы (Drag/Swipe).
*   **Группировка:** Объединяйте любые жесты в сложные **Сценарии**. Сценарии можно выстраивать в очередь для пакетного запуска.

### 2. 👁 Визуальная верификация (Интеллектуальный контроль)
Это главная особенность приложения. Кликер не просто нажимает в заданную точку, а *проверяет*, привело ли это к ожидаемому результату.
*   Перед выполнением шага кликер делает «снимок» (скриншот) экрана.
*   После клика он делает второй снимок и сравнивает их.
*   Если интерфейс отреагировал (открылось меню, загрузилась страница, изменилась кнопка) — шаг считается успешным. Если нет — система может подождать или пропустить шаг в зависимости от ваших настроек.

### 3. 📝 Продвинутый редактор и запись
*   **Плавающий виджет записи:** Записывайте сценарии "на лету" прямо поверх нужных приложений.
*   **Пошаговый редактор:** Любой записанный макрос можно открыть в редакторе, изменить координаты, задержки, длительность нажатий и переупорядочить шаги.
*   **Тестирование отдельного шага:** Вы можете протестировать работу одного конкретного клика/свайпа прямо из редактора, не запуская весь сценарий целиком.

### 4. 🔄 Импорт / Экспорт
Вы можете выгрузить созданные сценарии в JSON-формат для резервного копирования или чтобы поделиться с другими устройствами.

## ⚙️ Нюансы и тонкости настроек

Чтобы автокликер работал стабильно на любых устройствах, важно понимать механику его параметров. Настройки делятся на две категории: **Глобальные** и **Локальные (для каждого шага)**.

### Глобальные настройки (влияют на всё приложение)
*   **Визуальная верификация (Global Verification):** Главный рубильник «зрения». Если он выключен, кликер будет работать «вслепую» (как классический автокликер) и игнорировать настройки проверки у отдельных шагов. Требует разрешения на захват экрана (Media Projection).
*   **Задержка между действиями (Delay Between Actions):** Пауза по умолчанию между любыми двумя действиями, чтобы система успевала обработать анимации ОС.
*   **Восстановить приложение:** Возможность автоматически развернуть Prog Set Touch на экран после того, как автокликер отработал свои задачи в фоне.

### Локальные настройки (Индивидуально для каждого шага)
В редакторе сценариев, при нажатии на конкретный шаг, вам доступен режим тонкой настройки:

**Базовые настройки шага:**
1.  **Длительность (Duration, мс):** Как долго палец "удерживается" на экране. Для обычного клика оптимально `50-100 мс`. Для свайпов `300-600 мс`, чтобы система распознала это как плавное пролистывание, а не резкий рывок.
2.  **Задержка после шага (Step Delay, мс):** Увеличьте паузу только для этого конкретного шага, переопределив глобальную задержку.

**Настройки верификации (если включено "Проверять изменение экрана"):**
1.  **Таймаут (Timeout, мс):** Максимальное время ожидания визуального отклика (до 5 минут). Кликер делает снимки каждую секунду. Как только порог изменения достигнут, кликер мгновенно пойдет дальше.
2.  **Порог чувствительности (Threshold, %):** Процент пикселей на экране, который должен поменяться. Низкий порог (0.5% - 2%) подходит для небольших иконок. Высокий (10% - 30%) — для полного изменения экрана.
3.  **Продолжить при ошибке (Continue on Failure):** Самый важный флаг логики. Если включен: в случае, когда экран так и не изменился, кликер проигнорирует провал и перейдет к следующему шагу. Если выключен: при любой неудаче сценарий немедленно останавливается.

## 🚀 Рекомендации по использованию

1. **Не злоупотребляйте верификацией:** Скриншоты нагружают процессор. Включайте проверку экрана только на критически важных шагах.
2. **Используйте "Продолжить при ошибке" с умом:** Включайте этот флаг, если шаг направлен на закрытие всплывающего окна или рекламы, которой может и не быть на экране.
3. **Безопасность:** Если система блокирует захват экрана (в банковских приложениях или при вводе паролей срабатывает флаг защиты), экран на снимке будет черным, и верификация будет автоматически считаться успешной.
