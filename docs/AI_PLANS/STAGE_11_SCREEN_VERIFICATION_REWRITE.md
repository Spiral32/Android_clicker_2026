# План переработки: Логика визуальной верификации (Сравнение с эталоном)

## Цель
Изменить текущую логику проверки (которая проверяла "изменился ли экран вообще") на логику "соответствует ли экран сохраненному эталону (результату), который был зафиксирован во время записи".

## Фаза 1: Сохранение эталонов во время записи
- [x] **Хранилище скриншотов**:
   - Реализовать `ScreenshotStorageManager` (Kotlin), который будет сохранять `Bitmap` в локальную папку `app_data/screenshots/` (в сжатом виде, например WebP/JPEG).
   - В сущность `RecordedAction` и `ScenarioStep` добавить поле `resultImageFileName`.
- [x] **Модификация `RecorderManager`**:
   - Внедрить логику захвата "Результата" (Target Screenshot). Скриншот результата для шага `N` делается в момент старта шага `N+1`.
   - При нажатии "Стоп запись", делается финальный скриншот для самого последнего шага.
- [x] **Dart/Flutter**:
   - Обновить модели, парсеры `fromMap`/`toMap`.
   - При экспорте (Export) сценария конвертировать картинки в `Base64` и добавлять в JSON. При импорте — распаковывать обратно в файлы.
- [x] **Очистка памяти**:
   - При удалении сценария или записи, удалять связанные с ним файлы скриншотов через платформенный вызов.

## Фаза 2: Настройки сценария (UI)
- [x] **Flutter Слайдер Подобия**:
   - В `ScenarioStepEditorDialog` переосмыслить ползунок настройки. Теперь это "Требуемое подобие экрана" от 1% до 100% (Similarity Percent).

## Фаза 3: Движок выполнения (ExecutionEngine)
- [x] **Алгоритм сравнения (`ScreenshotVerifier`)**:
   - Реализовать функцию `compareWithTemplate(currentFrame, templateFileName, similarityThreshold)`.
   - Использовать метрику схожести (Histogram / pHash).
- [x] **Цикл выполнения (`ExecutionEngine`)**:
   - При старте шага: загрузить эталонную картинку (`resultImageFileName`).
   - Выполнить жест (тап/свайп).
   - В цикле (до `timeoutMs`):
     - Делать скриншот.
     - Сравнивать со скачанным эталоном.
     - Если `Similarity >= similarityThreshold` -> Успех (break).
     - Иначе -> Ждем 1 секунду, пробуем снова.
   - Учет `continueOnFailure`: Если после таймаута подобие не достигнуто, то если флаг включен — продолжаем, если выключен — останавливаемся.

## Фаза 4: Тестирование
- [ ] **Тест записи**: Проверить, что файлы картинок создаются и привязываются к правильным шагам.
- [ ] **Тест удаления**: Убедиться, что при удалении записи скриншоты физически удаляются из памяти.
- [ ] **Тест выполнения**: Проверить на тестовом приложении, что кликер корректно ждет появления ожидаемого экрана.
- [ ] **Тест экспорта/импорта**: Убедиться, что сценарии с картинками могут экспортироваться и импортироваться.
